# rag-knowledge

本技能让 Claude 可以调用企业 RAG 检索接口在知识库中查找相关资料。

## 使用指南

1. **明确 kb_id**：每次调用 `rag.search` 前必须确认当前是哪个知识库。如果用户没指定，先列出可用知识库（调 `rag.kb.list`）让用户选择。
2. **构造好查询**：query 应是用户问题的核心检索词，必要时改写成更检索友好的形式（去掉口语化表达）。
3. **解读结果**：返回的 chunks 含 chunk_id、heading_path、page_num、content。根据这些片段组织答案，并在每条断言后附 `[chunk_id=...]` 引用标签。
4. **零幻觉**：如果检索结果不足以回答问题，明确告知"知识库中未找到相关信息"，不要编造。

## 工具

- `rag.search`：在指定知识库中检索 top-K 相关 chunks（混合检索 + Reranker 精排）
- `rag.kb.list`：列出当前所有可用知识库

## 何时不使用

如果用户问题与企业内部资料无关（如天气、通用知识、数学计算），不要调用 RAG，直接用模型自己的知识回答。
